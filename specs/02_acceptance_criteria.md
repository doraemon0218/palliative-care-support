# 緩和ケア意思決定支援アプリ - 受け入れ基準

## AC-01 匿名入力
- [ ] 氏名、患者ID、住所など識別子フィールドが存在しない。
- [ ] 入力データ保存時に識別子が混入していないことを自動検証できる。

## AC-02 症状評価必須化
- [ ] 痛み評価未入力時に薬剤提案画面へ遷移できない。
- [ ] 未入力項目が視覚的に強調表示される。

## AC-03 標準薬剤提案
- [ ] 症状・重症度入力に応じて第一選択/代替薬が表示される。
- [ ] 各薬剤に禁忌・注意事項が表示される。

## AC-04 標準用量提案
- [ ] 患者条件に応じて初期用量レンジが変化する。
- [ ] 表示される用量に単位（mg/day 等）の欠落がない。

## AC-05 オピオイド換算
- [ ] 任意の基準ケースで、手計算とアプリ計算結果が一致する。
- [ ] 換算係数、式、丸め処理がUIで確認できる。
- [ ] 減量係数を適用した推奨量が別表示される。

## AC-06 投与経路変更タイミング
- [ ] 経口→注射切替ケースで、開始時刻と重複回避指示が表示される。
- [ ] 半減期に関する警告が条件に応じて表示される。

## AC-07 安全アラート
- [ ] 禁忌組み合わせ入力時に警告ダイアログが必ず表示される。
- [ ] 警告表示後、再確認ボタン押下まで確定操作ができない。

## AC-08 根拠トレーサビリティ
- [ ] すべての提案結果に根拠リンクと最終改訂日が付与される。

## AC-09 ログ監査
- [ ] 入力、計算、警告、最終確認が時系列で追跡できる。
- [ ] ログに個人特定情報が含まれない。

## AC-10 パフォーマンス
- [ ] 換算実行から結果表示まで1秒以内（95パーセンタイル）を満たす。

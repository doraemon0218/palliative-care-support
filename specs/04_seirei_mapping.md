# 聖隷三方原病院「症状緩和ガイド」アプリ化マッピング

対象ページ: [聖隷三方原病院 症状緩和ガイド](https://www.seirei.or.jp/mikatahara/doc_kanwa/)

## 1. 情報設計（IA）
Web目次をアプリ内で以下のナビゲーションに変換する。

- `症状から探す`
  - 疼痛
  - 消化器症状
  - 呼吸器症状
  - 精神症状
  - 全身症状
  - 死亡直前
- `実務ツール`
  - オピオイド換算
  - 経路変更タイミング計算
  - 代表オーダーテンプレート
- `付録・院内ルール`
  - デバイス仕様
  - 指示の出し方
  - 依頼方法
- `更新履歴`
  - 更新日、更新対象、要約

## 2. 画面構成（MVP）
### 2.1 ホーム
- 「症状選択」入口
- 「よく使うオーダー（オピオイド、鎮静薬）」ショートカット
- 「直近更新」カード（例: 不眠、下剤、せん妄更新）

### 2.2 症状詳細
- 各症状ごとに以下をタブ表示
  - 評価（必須チェックリスト）
  - 第一選択/代替薬
  - 用量・投与法
  - 注意点・禁忌
  - 根拠（元ガイドへのリンクと改訂日）

### 2.3 計算ツール
- オピオイド等価換算
- レスキュー量目安
- 経口/注射/貼付の切替タイミング支援

### 2.4 監査ビュー
- 入力値
- 算出根拠（係数、式、丸め）
- 警告履歴
- 最終確認者ロール

## 3. ページ内容から機能への変換ルール
### 3.1 「読む情報」を「入力→出力フロー」に変換
- 例: 「疼痛(1) 持続的疼痛」ページ
  - 入力: 痛み強度、現行オピオイド、レスキュー回数、腎機能
  - 出力: 推奨薬、初期用量、レスキュー量、再評価タイミング

### 3.2 「処方例」を「安全付きオーダー案」に変換
- 単なる処方文ではなく、禁忌チェックと上限アラートを追加。

### 3.3 「付録」を「意思決定補助メタ情報」に変換
- デバイス仕様、PCA運用ルール、依頼方法は計算結果の補助情報として文脈表示。

## 4. 主要ユースケース（サイト目次ベース）
- 疼痛: 等価換算を伴う初期設定・切替
- 呼吸困難: 原因別の標準薬剤選択
- せん妄・不眠: 更新頻度の高い処方例をガード付きで提示
- 死亡直前: 鎮静や死前喘鳴対応での迅速オーダー支援

## 5. コンテンツ運用要件
- 各コンテンツに `最終更新日` と `版番号` を保持
- 改訂時は差分を記録し、アプリ内更新履歴に自動反映
- 「標準ではない可能性」や「保険適用注意」を警告テキストとして固定表示

## 6. 安全要件（ページ注意書きを反映）
- 本アプリの表示内容は「臨床判断支援」であり、最終判断は主治医責任であることを常時表示。
- 保険適用外リスクのある提案には明示的な注意喚起を付ける。
- 院内運用ルールを優先し、施設設定で上書き可能にする。

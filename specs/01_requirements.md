# 緩和ケア意思決定支援アプリ - 要件定義

## 1. スコープ
### 1.1 対象
- 症状評価支援
- 標準薬剤選択支援
- 標準投与量提案
- オピオイド換算
- 投与経路変更時のタイミング提案

### 1.2 非対象（初期）
- 電子カルテへの自動オーダー投入
- 患者同定情報の保存

## 2. 機能要件（FR）
### FR-01: 匿名患者コンテキスト入力
- 年齢層、体重、腎機能区分、肝機能区分、現在症状、現行投薬情報を入力できる。
- 氏名・IDなど個人特定情報は入力不可またはマスク。

### FR-02: 症状評価フロー
- 症状別に評価項目（例: 痛み、呼吸困難、嘔気、せん妄）を順に提示。
- 必須評価が未完了の場合、薬剤提案に進めない。

### FR-03: 標準薬剤候補提示
- 症状・重症度・禁忌情報から第一選択/代替選択を提示。
- 選択理由、禁忌、注意事項を併記。

### FR-04: 標準用量提案
- 薬剤、患者条件、投与経路に基づき初期用量レンジを提示。
- 最小用量から開始する推奨フラグを持つ。

### FR-05: オピオイド換算
- 入力: 現在薬剤、1日総投与量、投与経路、レスキュー回数。
- 出力: 等価用量、減量係数適用後推奨量、レスキュー量の目安。
- 換算根拠（換算係数、式、丸めルール）を必ず表示。

### FR-06: 投与方法変更タイミング支援
- 経口→注射、貼付→経口など変更パターンごとに開始時刻と重複回避指示を提示。
- 半減期・剤形特性に基づく注意喚起を出す。

### FR-07: 安全アラート
- しきい値超過、禁忌組み合わせ、換算結果の異常値を警告。
- 警告時は「再確認」操作を必須化。

### FR-08: 根拠リンク
- 提示内容ごとにマニュアル根拠と最終改訂日を表示。

### FR-09: 操作ログ
- 入力値、計算結果、警告表示、最終確認操作を匿名ログとして記録。

## 3. 非機能要件（NFR）
### NFR-01: 可用性
- 病棟業務時間帯で99.9%以上の稼働を目標。

### NFR-02: 性能
- 主要計算（換算/用量/タイミング）は入力確定後1秒以内に結果表示。

### NFR-03: セキュリティ
- 患者個人情報を保存しない設計。
- 通信・保存データは暗号化。

### NFR-04: 監査性
- 計算ロジックのバージョン、薬剤テーブルの版管理を実施。

### NFR-05: 保守性
- 換算係数や薬剤標準量は設定データとして更新可能。

## 4. ユースケース（抜粋）
### UC-01: 痛みコントロールでの経口モルヒネから持続皮下注へ切替
1. 匿名患者情報を入力
2. 現行1日総量とレスキュー使用量を入力
3. アプリが等価換算と減量後用量を提示
4. 開始タイミングと重複回避指示を表示
5. 医師/看護師が再確認し適用

### UC-02: 呼吸困難に対する標準薬剤選択
1. 症状重症度を評価
2. 禁忌有無を選択
3. 第一選択と代替薬、初期用量が提示
4. 根拠参照後に意思決定
